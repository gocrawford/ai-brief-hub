# X.com — Social Intelligence

## Mission

Capture the AI-discourse signals from X. Two streams:
1. **Curated voices** — researchers, lab leaders, builders whose posts move the discourse.
2. **Viral detection** — high-engagement posts on AI topics from accounts not on the curated list. This is how we catch breakouts and discover new voices to add to the curated list.

## Viral detection sweep (every week, regardless of curated list)

Run topic searches via `browser_task` against X search. **Engagement floor: ≥ 5,000 likes OR ≥ 1,000,000 views OR ≥ 1,000 reposts.** Surface qualifying posts even when the author is not on the curated list.

Topic queries:
- `"claude code"` recent
- `"agent loop"` OR `"agent harness"` recent
- `"MCP"` OR `"model context protocol"` recent
- `"computer use"` AI recent
- `"AI Act"` OR `"EU AI"` OR `"NIST AI"` recent
- `"reasoning model"` OR `"thinking model"` recent
- `"open source AI"` OR `"open weights"` recent
- New model name as it emerges (e.g., when Claude Mythos / GPT-X / Gemini-X is rumored or released)

When a viral author keeps appearing across weeks, add them to the curated list via Suggested Refinements.

## Method

1. For each curated account, use `browser_task` to fetch the user's profile filtered to the week's date range. Capture posts with engagement above the account's median.
2. Run the viral detection sweep separately. Tag those items with `source_via: "viral_sweep"` so we can track what's coming from outside the curated list.
3. For each candidate, verify the URL resolves to the actual post and the text matches what's quoted.
4. Threads: include the head post URL; summarize the full thread.
5. Distinguish announcements (Tool Announcements section, with priority badges) from insights (Key Insights section, with `key_takeaways`).
6. Hot Takes/Debates: include only when there's substantive disagreement with a clear position on each side.

## Freshness & recurrence

Set `freshness.target_pct_new = 0.80` and `min_items = 15`. Against last week's `data/<prev_week>/x.json`:

- **Same post** — exclude entirely.
- **Same author, new post on the same topic** — `recurrence: "returning"` if it was a Top Pick last week.
- **Same topic, multiple new posts** — fold into `emerging_signals` or treat as a debate cluster.
- Empty section line: `"No major new developments this week."`

## Output structure (`x.json`)

`tab_id: "x"`, `subtitle: "Social Intelligence"`, `icon: "Twitter"`.

**Sections:**
1. `executive_summary` — 3–5 bullets
2. `top_picks` — 5–8 posts
3. `tool_announcements` — Table-style. priority: CRITICAL / HIGH / MEDIUM
4. `key_insights` — Threads with takeaways. `enterprise_translation` if it's a workflow pattern.
5. `viral_breakouts` — Section dedicated to viral-sweep items not from curated accounts. Always include source/handle so the reader can follow up.
6. `hot_takes` — 0–3 substantive debates
7. `emerging_signals` — Cross-account patterns
8. `index` — All posts considered, **min 15 rows** (`min_items: 15`)
9. `suggested_refinements` — Specifically: "Accounts to Add" (from viral sweep showing repeated signal), "Accounts to Drop" (curated accounts that produced nothing useful for 4+ weeks)

## Per-item required fields

- `title` (post excerpt or quote), `author` (display name), `source` (@handle), `date`, `primary_url` (canonical x.com URL), `summary`, `score`, `verified`, `verified_at`
- Engagement: likes, comments, shares (`engagement` object) — captured at fetch time
- Top Picks add: `key_takeaways`, `suggested_actions`, `enterprise_translation` if applicable

## What to drop

- Quote-tweets that add no new content
- Subtweets / drama with no technical content
- Engagement-bait threads from prompt-engineering grifters
- Posts whose URL no longer resolves
- Viral posts that on inspection are jokes, screenshots without context, or AI-generated rage-bait
