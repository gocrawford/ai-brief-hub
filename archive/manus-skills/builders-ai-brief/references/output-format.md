# Builders Brief — Output Format Reference

Use these exact section templates when writing the weekly Builders AI Intelligence Brief.

## Header

```markdown
# Weekly Builders AI Intelligence Brief
**Date:** [Date Range]
**Prepared for:** Bill & Team
**Focus:** Real AI Workflows & Automation Systems

---
```

## Section 1: Executive Summary

```markdown
## 1. Executive Summary
```

Opening paragraph (2-3 sentences on the week's overarching theme), followed by:

```markdown
[Opening paragraph: 2-3 sentences on the week's overarching theme in AI building.]

**Top 3 Builder Stories This Week:**
1. **[Story Title]:** [2 sentence summary with reference number.]
2. **[Story Title]:** [2 sentence summary with reference number.]
3. **[Story Title]:** [2 sentence summary with reference number.]

**Key Takeaway for the Team:** [1-2 sentences on the meta-pattern Bill's team should internalize.]
```

## Section 2: Top Picks

```markdown
## 2. Top Picks
```

Top 3-4 highest-scoring implementations:

```markdown
### [Project/Article Title]
**Source:** [Platform] [ref]
**Score:** [X.X]/10

**The Workflow:**
[2-3 paragraph description of what was built, the architecture, and how it works.]

**Measurable Results:**
[Specific metrics: time saved, accuracy, cost reduction, etc. Include reference number.]

**How Bill's Team Could Adapt This:**
[1-2 paragraphs with specific recommendations for adapting this to Bill's tool stack and enterprise context.]
```
